# Dynamic Data Conversion Plan

## Overview
The TruthLens project contains significant amounts of hardcoded data that should be converted to dynamic, realtime fetching for better maintainability, scalability, and real-time updates. This plan outlines the systematic conversion of hardcoded data to database-driven or API-fetched content.

## Issues Identified

### 1. **Homepage Static Content** (`src/app/page.tsx`)
- **heroCards**: Static metrics cards (Trust Score, Source Network, Bias Detection, etc.)
- **howItWorks**: Static workflow steps
- **trustMetrics**: Static performance metrics

### 2. **Learning Modules** (`src/app/learn/page.tsx`)
- **modules**: Hardcoded learning articles with titles, descriptions, images, dates
- **categories**: Static category filters

### 3. **Mock API Data**
- **Submissions API**: Returns hardcoded mock submissions instead of real database data
- **Notifications API**: Returns hardcoded notifications instead of real user notifications
- **Missing real database integration**

### 4. **Static Image URLs**
- **83 hardcoded Googleusercontent image URLs** across multiple files
- **No centralized image management**
- **No fallback handling for broken images**

### 5. **Configuration Constants**
- **Source categories/tiers**: Hardcoded arrays in multiple admin components
- **Validation constants**: Hardcoded in API routes

## Implementation Plan

### Phase 1: Database Schema Extensions
**Goal**: Create tables for dynamic content management

1. **Homepage Content Table**
   ```sql
   CREATE TABLE homepage_content (
     id SERIAL PRIMARY KEY,
     section VARCHAR(50) NOT NULL, -- 'hero_cards', 'how_it_works', 'trust_metrics'
     title VARCHAR(255),
     value VARCHAR(100),
     description TEXT,
     icon VARCHAR(100),
     accent VARCHAR(100),
     sort_order INTEGER DEFAULT 0,
     is_active BOOLEAN DEFAULT true,
     created_at TIMESTAMP DEFAULT NOW(),
     updated_at TIMESTAMP DEFAULT NOW()
   );
   ```

2. **Learning Modules Table**
   ```sql
   CREATE TABLE learning_modules (
     id SERIAL PRIMARY KEY,
     title VARCHAR(500) NOT NULL,
     description TEXT,
     tag VARCHAR(50),
     tag_color VARCHAR(100),
     duration VARCHAR(50),
     published_date DATE,
     image_url VARCHAR(1000),
     content_url VARCHAR(1000),
     sort_order INTEGER DEFAULT 0,
     is_active BOOLEAN DEFAULT true,
     created_at TIMESTAMP DEFAULT NOW(),
     updated_at TIMESTAMP DEFAULT NOW()
   );
   ```

3. **System Metrics Table**
   ```sql
   CREATE TABLE system_metrics (
     id SERIAL PRIMARY KEY,
     metric_key VARCHAR(100) UNIQUE NOT NULL,
     value VARCHAR(255) NOT NULL,
     label VARCHAR(255),
     last_updated TIMESTAMP DEFAULT NOW(),
     update_frequency INTERVAL -- how often to refresh
   );
   ```

### Phase 2: Image Management System
**Goal**: Centralized image storage and management

1. **Image Assets Table**
   ```sql
   CREATE TABLE image_assets (
     id SERIAL PRIMARY KEY,
     filename VARCHAR(255) NOT NULL,
     original_url VARCHAR(1000),
     local_path VARCHAR(500),
     alt_text VARCHAR(500),
     category VARCHAR(100), -- 'learning', 'ui', 'icons', etc.
     width INTEGER,
     height INTEGER,
     file_size INTEGER,
     uploaded_at TIMESTAMP DEFAULT NOW()
   );
   ```

2. **Create Image Upload API**
   - `/api/admin/upload-image`
   - Support for bulk uploads
   - Automatic optimization and CDN storage

3. **Image Service**
   - Centralized image fetching with fallbacks
   - Lazy loading and optimization
   - Error handling for broken images

### Phase 4: Frontend Component Updates

1. **Homepage Component Refactor**
   ```typescript
   // src/app/page.tsx
   export default function Home() {
     const [content, setContent] = useState(null);
     const [loading, setLoading] = useState(true);
     
     useEffect(() => {
       fetch('/api/content/homepage')
         .then(res => res.json())
         .then(setContent)
         .finally(() => setLoading(false));
     }, []);
     
     if (loading) return <LoadingSpinner />;
     
     return (
       <main>
         <HeroSection cards={content.heroCards} />
         <HowItWorks steps={content.howItWorks} />
         <TrustMetrics metrics={content.trustMetrics} />
       </main>
     );
   }
   ```

2. **Learning Page Refactor**
   ```typescript
   // src/app/learn/page.tsx
   export default function LearnPage() {
     const [modules, setModules] = useState([]);
     const [categories, setCategories] = useState([]);
     
     useEffect(() => {
       fetch('/api/content/learning-modules')
         .then(res => res.json())
         .then(data => {
           setModules(data.modules);
           // Extract unique categories from modules
           const uniqueCategories = [...new Set(data.modules.map(m => m.category))];
           setCategories(uniqueCategories);
         });
     }, []);
   }
   ```

### Phase 6: Migration Strategy

1. **Data Migration Script**
   ```typescript
   // scripts/migrate-hardcoded-data.ts
   async function migrateHomepageData() {
     const heroCards = [
       // Current hardcoded data from page.tsx
     ];
     
     await db.homepage_content.createMany({
       data: heroCards.map(card => ({
         section: 'hero_cards',
         title: card.title,
         value: card.value,
         description: card.note,
         icon: card.icon,
         accent: card.accent,
         sort_order: 0,
         is_active: true
       }))
     });
   }
   ```

2. **Gradual Rollout**
